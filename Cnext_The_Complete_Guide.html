Preface — About This Book

This book is your complete guide to learning the Cnext programming language, from absolute beginner to confident practitioner.

Cnext is a modern, high-level, statically-typed programming language that transpiles to C. It was designed to give developers the best of both worlds: the modern language features you love (generics, closures, pattern matching, async/await, OOP) combined with the raw performance and portability of C.

The compiler itself is written in C (GNU C11 standard) and produces equivalent C code that is then compiled by GCC or Clang into native executables.

Who This Book Is For

  • New programmers who want a clean, modern first language
  • C developers looking for a higher-level abstraction without sacrificing performance
  • Developers from other languages (Python, JavaScript, Rust, Go, Swift) who want to explore systems programming
  • Students studying compiler design and language theory

How to Use This Book

Read the chapters in order for a structured learning experience, or jump to specific topics using the Table of Contents. Every chapter includes code examples you can type, run, and modify.

Conventions Used

  • Code — inline code for keywords, functions, and file names
  • Bold — important terms and definitions
  • Italic — emphasis and book titles
  • Code blocks — complete, runnable examples
Tip: All code examples in this book use the .cn file extension and can be run with cnext run filename.cn.

Chapter 1 — Introduction to Cnext

Understand what Cnext is, why it exists, and how it compares to other languages.

1.1 What is Cnext?

Cnext is a transpiler — a program that translates source code from one language to another. Specifically, Cnext takes .cn source files and produces equivalent .c files, which are then compiled by GCC or Clang into native machine code.

The name "Cnext" reflects its philosophy: C, but next generation. It preserves everything that makes C powerful (performance, portability, low-level control) while adding modern conveniences that make developers more productive.

1.2 Key Features

FeatureDescription
Static TypingType safety with optional type inference via var
Object-Oriented ProgrammingClasses, inheritance, interfaces, and traits
GenericsParameterized types with monomorphization (zero-cost abstractions)
ClosuresFirst-class functions with variable capture
Pattern MatchingPowerful match expressions for enums and values
Error Handlingtry/catch/finally with throw
GeneratorsLazy iteration with yield
Async/AwaitAsynchronous programming with cooperative scheduling
ThreadingBuilt-in Mutex and Channel primitives
C InteroperabilityDirect FFI with extern "C"
MacrosCompile-time code generation
Optional SemicolonsWrite cleaner code without mandatory semicolons
Built-in ToolingFormatter, linter, REPL, doc generator, package manager
Cross-PlatformLinux, macOS, and Windows support

1.3 How Cnext Works

The Cnext compiler follows a classic multi-stage pipeline:

Source (.cn) → Lexer → Tokens → Parser → AST → Semantic Analyzer → Optimizer → Code Generator → C Source (.c) → GCC/Clang → Executable
  1. Lexer — Reads your .cn source code and breaks it into tokens (keywords, identifiers, literals, operators)
  2. Parser — Parses the token stream into an Abstract Syntax Tree (AST)
  3. Semantic Analyzer — Performs type checking, scope resolution, and safety validation
  4. Optimizer — Applies 9 optimization passes (constant folding, dead code elimination, tail-call detection, etc.)
  5. Code Generator — Traverses the AST and emits equivalent C code
  6. C Compiler — GCC or Clang compiles the generated C into a native executable

1.4 Why Cnext?

If you know C, you know its strengths and weaknesses. Cnext addresses the weaknesses while preserving the strengths:

C Pain PointCnext Solution
No type inferencevar keyword for automatic type deduction
No OOPFull class system with inheritance, interfaces, and traits
Manual memory managementChecked allocation wrappers, arena/pool allocators
No genericsFull generic system with monomorphization
Fragile error handlingStructured try/catch/finally
No closuresFirst-class closures with variable capture
No built-in package managerIntegrated package registry and dependency management
Inconsistent formattingBuilt-in code formatter

1.5 Cnext vs. Other Languages

LanguageCnext Comparison
CCnext transpiles to C; same performance, more features
RustSimilar goals (safety + performance), but Cnext is simpler and transpiles to C
GoBoth emphasize simplicity; Cnext has richer OOP and closures
SwiftSimilar syntax feel; Cnext targets systems programming
TypeScriptSimilar transpilation approach; Cnext targets native code, not JavaScript

Chapter 2 — Installation & Setup

Get Cnext installed on your system in minutes.

2.1 System Requirements

  • A C compiler: GCC or Clang (C11 standard)
  • Make (Linux/macOS) or MinGW/MSYS2 (Windows)
  • libcurl (Linux/macOS only, for networking features)
  • Git (for building from source)

2.2 Quick Install (One Command)

Linux / macOS

curl -fsSL https://raw.githubusercontent.com/Melton122/cnext/main/install.sh | bash

Windows (PowerShell)

iwr -useb https://raw.githubusercontent.com/Melton122/Cnext/main/install.ps1 | iex

Or download install.bat from the GitHub Releases page and double-click it.

2.3 Build from Source

git clone https://github.com/Melton122/cnext.git
cd cnext
make
sudo make install

Build Options

CommandDescription
makeRelease build (optimized)
make DEBUG=1Debug build with symbols
make cleanClean build artifacts
make installInstall to /usr/local/bin
make testRun the built-in test suite
make formatFormat source code
make helpShow all available targets

2.4 Verify Installation

cnext version

Expected output:

Cnext version 9.0.0

2.5 Run a System Check

cnext doctor

This checks that all required dependencies (GCC, Make, libcurl) are installed and available.

2.6 Troubleshooting

"gcc: command not found"

  • Linux: sudo apt install gcc
  • macOS: xcode-select --install
  • Windows: Install MSYS2 or MinGW

"runtime.h: No such file or directory"

The include/ directory was not installed. Reinstall with sudo make install or manually copy the include files.

Permission Denied

chmod +x cnext

Windows Defender Warning

Click "More info" then "Run anyway" if Windows SmartScreen blocks the executable.

Chapter 3 — Your First Program

Write, compile, and run your very first Cnext program.

3.1 Hello, World!

Create a file called hello.cn:

main {
    printin("Hello, World!")
}

Run it:

cnext run hello.cn

Output:

Hello, World!

Congratulations! You just wrote and ran your first Cnext program.

3.2 Understanding the Code

Let's break down what happened:

  • main { ... } — The entry point of every Cnext program. This is equivalent to int main(void) { ... } in C.
  • printin("Hello, World!") — Prints the string followed by a newline. This maps to printf() in the generated C code.

3.3 Creating a Project

For larger programs, use cnext new to create a project scaffold:

cnext new my-project

This creates:

  • A cnext.toml configuration file
  • A src/ directory with a starter file
  • VS Code configuration files
  • A README file

3.4 Run vs. Build

CommandDescription
cnext run hello.cnCompile and run immediately (no executable saved)
cnext build hello.cnCompile to an executable file
cnext build hello.cn --releaseOptimized build with -O2
cnext build hello.cn --debugDebug build with -O0 -g

3.5 Using the REPL

Cnext includes an interactive REPL (Read-Eval-Print Loop):

cnext repl

Type expressions and see results immediately:

> 2 + 2
4
> "hello" + " world"
hello world
> var x = 10
> x * 2
20

Chapter 7 — Control Flow

Direct the execution of your programs with conditionals, loops, and pattern matching.

7.1 If/Else

var x = 15

if x > 10 {
    printin("big")
} else if x > 5 {
    printin("medium")
} else {
    printin("small")
}

7.2 While Loop

var i = 0
while i < 10 {
    printin(i)
    i = i + 1
}

7.3 For Loop

for int i = 0; i < 10; i = i + 1 {
    printin(i)
}

7.4 For-In Loop

int[] nums = {1, 2, 3, 4, 5}
for var n in nums {
    printin(n)
}

7.5 Break and Continue

for int i = 0; i < 10; i = i + 1 {
    if i == 3 {
        continue  // skip 3
    }
    if i == 7 {
        break     // stop at 7
    }
    printin(i)
}

7.6 Switch

var x = 2
switch x {
    case 1:
        printin("one")
    case 2:
        printin("two")
    case 3:
        printin("three")
    default:
        printin("other")
}

7.7 Match (Pattern Matching)

enum Color { RED, GREEN, BLUE }

Color c = RED
match c {
    RED => printin("red")
    GREEN => printin("green")
    BLUE => printin("blue")
    _ => printin("other")
}

7.8 Guard Clauses

func process(int? value) {
    guard value != null else {
        printin("No value provided")
        return
    }
    // value is guaranteed non-null here
    printin("Processing: " + value)
}

Chapter 8 — Functions

Organize your code into reusable, composable functions.

8.1 Basic Functions

func greet(str name) {
    printin("Hello, " + name + "!")
}

greet("World")

8.2 Return Values

func add(int a, int b) -> int {
    return a + b
}

var result = add(5, 3)
printin(result)  // 8

8.3 Arrow Functions

For single-expression functions, use the arrow syntax:

func multiply(int a, int b) -> int => a * b
func square(int x) -> int => x * x

printin(multiply(3, 4))  // 12
printin(square(5))       // 25

8.4 Default Arguments

func greet(str name, str greeting = "Hello") {
    printin(greeting + ", " + name + "!")
}

greet("World")                    // Hello, World!
greet("Alice", greeting = "Hi")   // Hi, Alice!

8.5 Named Arguments

func create_person(str name, int age, str city = "Unknown") {
    printin(name + " is " + age + " from " + city)
}

create_person(name = "Bob", age = 25, city = "NYC")
create_person(age = 30, name = "Charlie")

8.6 Variadic Functions

Accept any number of arguments with ...:

func sum(int... args) -> int {
    var total = 0
    for var n in args {
        total = total + n
    }
    return total
}

printin(sum(1, 2, 3))          // 6
printin(sum(1, 2, 3, 4, 5))    // 15

8.7 Higher-Order Functions

Functions can be passed as arguments and returned:

func apply(int x, func f) -> int {
    return f(x)
}

func double(int x) -> int => x * 2
func square(int x) -> int => x * x

printin(apply(5, double))  // 10
printin(apply(5, square))  // 25

8.8 Extension Methods

Add methods to existing types:

extend str {
    func shout() -> str {
        return self + "!!!"
    }
}

var msg = "Hello"
printin(msg.shout())  // Hello!!!

8.9 Operator Overloading

class Vec2 {
    int x
    int y

    func new(int x_val, int y_val) {
        self.x = x_val
        self.y = y_val
    }

    func operator+(Vec2 other) -> Vec2 {
        return new Vec2(self.x + other.x, self.y + other.y)
    }
}

var a = new Vec2(1, 2)
var b = new Vec2(3, 4)
var c = a + b
printin(c.x)  // 4
printin(c.y)  // 6

class Circle {
    float radius

    func new(float r) {
        self.radius = r
    }

    func area() -> float {

class Animal {

}

class Dog extends Animal {
    func new(str n) {
        super.new(n)
    }

    override func speak() {
        printin("Woof!")
    }
}

Dog d = new Dog("Rex")
d.speak()  // Woof!

12.4 Null Safety with Classes

var int_box = new Box<int>(42)

printin(int_box.get())  // 42
printin(str_box.get())  // "hello"

Chapter 13 — Interfaces & Traits

Define contracts and share behavior across unrelated types.

13.1 Interfaces

Interfaces define a set of methods that a class must implement:

interface Drawable {
    func draw()
}

class Circle implements Drawable {
    float radius

    func new(float r) {
        self.radius = r
    }

    override func draw() {
        printin("Drawing circle with radius " + self.radius)
    }
}

13.2 Traits

Traits provide default implementations along with the contract:

trait Loggable {
    func log() {
        printin("Logging: " + self.to_string())
    }
}

class User implements Loggable {
    str name

    func new(str n) {
        self.name = n
    }

    func to_string() -> str {
        return "User(" + self.name + ")"
    }
}

var user = new User("Alice")
user.log()  // Logging: User(Alice)

13.3 Generic Interfaces

interface Container<T> {
    func add(T item)
    func get(int index) -> T
    func size() -> int
}

class List<T> implements Container<T> {
    T[] items

    func new() {
        self.items = {}
    }

    override func add(T item) {
        // Add item to list
    }

    override func get(int index) -> T {
        return self.items[index]
    }

    override func size() -> int {
        return self.items.length
    }
}

13.4 Generic Traits

trait Serializable<T> {
    func serialize() -> str
    func deserialize(str data) -> T
}

class User implements Serializable<User> {
    str name

    func new(str n) {
        self.name = n
    }

    override func serialize() -> str {
        return '{"name": "' + self.name + '"}'
    }

    override func deserialize(str data) -> User {
        return new User("parsed")
    }
}

Chapter 14 — Closures

Work with first-class functions that capture variables from their enclosing scope.

14.1 Basic Closures

var x = 10
var add_x = (int a) => a + x
printin(add_x(5))  // 15

14.2 Capturing Variables

Closures capture variables by reference and can modify them:

var counter = 0
var increment = () => {
    counter = counter + 1
    return counter
}

printin(increment())  // 1
printin(increment())  // 2
printin(increment())  // 3

14.3 Closures as Arguments

func apply(int x, func f) -> int {
    return f(x)
}

func double(int x) -> int => x * 2
func square(int x) -> int => x * x

printin(apply(5, double))  // 10
printin(apply(5, square))  // 25

14.4 Returning Closures

func make_adder(int x) -> func {
    return (int a) => a + x
}

var add5 = make_adder(5)
var add10 = make_adder(10)

printin(add5(3))   // 8
printin(add10(3))  // 13

14.5 Closures in Loops

Warning: Be careful with closures inside loops. Create a new variable for each iteration to capture the correct value. 
var functions = {}
for int i = 0; i < 5; i = i + 1 {
    var val = i  // New variable per iteration
    functions[i] = () => val
}

printin(functions[0]())  // 0
printin(functions[1]())  // 1

14.6 Array Operations with Closures

int[] nums = {1, 2, 3, 4, 5}

// Filter

Chapter 17 — Generators & Coroutines

17.1 Basic Generators

Generators produce values lazily using yield:

func count_to(int n) -> iter<int> {
    var i = 1
    while i <= n {
        yield i
        i = i + 1
    }
}

for var x in count_to(5) {
    printin(x)  // 1, 2, 3, 4, 5
}

17.2 Fibonacci Sequence

func fibonacci() -> iter<int> {
    var a = 0, b = 1
    while true {
        yield a
        var temp = a + b
        a = b
        b = temp
    }
}

for var x in fibonacci() {
    if x > 100 { break }
    printin(x)
}

17.3 Generator with Parameters

        i = i + 1

    yield 2

17.5 Coroutines

Coroutines are generators that support bidirectional communication:

printin(gen.next())  // 1
printin(gen.next())  // 2

    var m = mutex_new()

    func increment() {
        mutex_lock(m)

    channel_send(ch, "message3")

19.6 Thread Safety Tips

  1. Always use mutex when accessing shared data
  2. Keep critical sections as small as possible
  3. Always unlock mutex, even on error paths
  4. Use channels for communication, mutex for shared state
  5. Avoid deadlocks by locking in consistent order

Chapter 26 — Range-Based For Loops

Cnext v10 introduces the range operator .. and range-based for loops for elegant iteration over integer ranges.

The Range Operator

The .. operator creates an exclusive range from a start value to an end value:

// Range expression: 0..5 means [0, 1, 2, 3, 4]
// The end value is EXCLUSIVE (not included)
var r = 0..5

Range-Based For Loops

Use for with in and a range expression to iterate over integer ranges:

// Explicit type
for int i in 0..5 {
    printin(i)  // 0, 1, 2, 3, 4
}

// Inferred type with var
for var j in 1..10 {
    printin(j)  // 1, 2, 3, ..., 9
}

// Calculate sum of 1 to 100
int sum = 0
for int i in 1..101 {
    sum = sum + i
}
printin("Sum: ", sum)  // 5050
Tip: The range operator is exclusive on the right side. 0..5 produces values 0, 1, 2, 3, 4. To include the end value, add 1: 0..6 produces 0 through 5.

    }
    printin("Scope exited")
}

// Output:
// Opening resource
// Using resource
// Closing resource     <-- deferred
// Scope exited

Appendix A — Complete Grammar Reference

The formal EBNF grammar of the Cnext language.

Program Structure

program        = { declaration } ;
declaration    = func_decl | class_decl | interface_decl | trait_decl
               | enum_decl | struct_decl | macro_decl | statement ;

Functions

func_decl      = [ "async" ] "func" IDENTIFIER "(" [ params ] ")" [ "->" type ] block ;
params         = param { "," param } ;
param          = type IDENTIFIER [ "=" expression ] | "..." IDENTIFIER ;
arrow_func     = [ "async" ] "func" IDENTIFIER "(" [ params ] ")" "->" type "=>" expression ;

Classes

class_decl     = "class" IDENTIFIER [ "<" type_params ">" ]
                 [ "extends" type ] [ "implements" type_list ] "{" { member } "}" ;
member         = field | method | constructor ;
field          = type IDENTIFIER ;
method         = func_decl ;
constructor    = "func" "new" "(" [ params ] ")" block ;

Statements

statement      = expr_stmt | block | if_stmt | while_stmt | for_stmt
               | for_in_stmt | return_stmt | throw_stmt | try_stmt
               | var_decl | const_decl | switch_stmt | match_stmt
               | guard_stmt ;
if_stmt        = "if" expression block { "else if" expression block } [ "else" block ] ;
while_stmt     = "while" expression block ;
for_stmt       = "for" var_decl ";" expression ";" expression ";" block ;
for_in_stmt    = "for" "var" IDENTIFIER "in" expression block ;
return_stmt    = "return" [ expression ] ;
throw_stmt     = "throw" expression ;
try_stmt       = "try" block { "catch" "(" type IDENTIFIER ")" block }
                 [ "finally" block ] ;
switch_stmt    = "switch" expression "{" { "case" expression ":" block } [ "default:" block ] "}" ;
match_stmt     = "match" expression "{" { pattern "=>" expression } [ "_ =>" expression ] "}" ;
guard_stmt     = "guard" expression "else" block ;

Expressions

expression     = assignment ;
assignment     = IDENTIFIER "=" assignment | ternary ;
ternary        = logical_or [ "?" expression ":" expression ] ;
logical_or     = logical_and { "||" logical_and } ;
logical_and    = equality { "&&" equality } ;
equality       = comparison { ("==" | "!=") comparison } ;
comparison     = addition { ("<" | ">" | "<=" | ">=") addition } ;
addition       = multiply { ("+" | "-") multiply } ;
multiply       = unary { ("*" | "/" | "%") unary } ;
unary          = ("-" | "!" | "await") unary | postfix ;
postfix        = primary { "[" expression "]" | "." IDENTIFIER | "(" [ args ] ")" } ;
primary        = NUMBER | STRING | CHAR | "true" | "false" | "null"
               | IDENTIFIER | "(" expression ")"
               | array_literal | tuple_literal | lambda ;

Types

type           = primitive | array_type | tuple_type | function_type | generic_type ;
primitive      = "int" | "long" | "float" | "double" | "str" | "bool" | "char"
               | "byte" | "uint" | "ulong" | "ushort" | "ubyte" ;
array_type     = type "[]" ;
tuple_type     = "(" type { "," type } ")" ;
function_type  = "func" "(" [ type_list ] ")" [ "->" type ] ;
generic_type   = IDENTIFIER "<" type { "," type } ">" ;
nullable_type  = type "?" ;
