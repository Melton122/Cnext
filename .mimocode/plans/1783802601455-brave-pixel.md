# Cnext v3.5 — Closures & Async Implementation Plan

## Current Status
- **Closures**: DONE (reference capture via `#define` macros + env structs)
- **Generators/Yield**: DONE (state machine with `_gen_*_frame` + `_gen_*_next()`)
- **Async/Await**: NOT DONE
- **Coroutines**: NOT DONE (generators are stackless coroutines, but no general coroutine API)
- **Multithreading**: NOT DONE

## Feature 1: Generalized Coroutines

Extend the existing generator state machine to support general coroutines.

### Language Syntax
```
// Coroutine: can yield values AND be resumed with values
coroutine func producer() -> iter<int> {
    var i = 0
    while true {
        var received = yield i   // yield value out, receive value in
        i = i + received
    }
}

// Usage
var co = producer()
var val1 = resume co           // first yield
var val2 = resume co with 10   // resume with value
```

### Implementation
- Add `TOKEN_COROUTINE`, `TOKEN_RESUME`, `TOKEN_WITH` keywords to lexer
- Add `AST_COROUTINE_DECL`, `AST_RESUME_EXPR` to AST
- Extend generator codegen to support `yield` returning a value (via `f->_received` field in frame)
- Add `resume` expression that calls `_gen_*_next()` and sets `f->_received`

### Files to modify
- `include/lexer.h` — add tokens
- `src/lexer.c` — recognize keywords
- `include/ast.h` — add AST nodes
- `src/parser.c` — parse syntax
- `src/semantics.c` — validate usage
- `src/codegen.c` — extend generator frame, generate resume code
- `include/runtime.h` — no changes needed (existing infrastructure works)

## Feature 2: Async/Await

Cooperative async/await built on top of the coroutine state machine.

### Language Syntax
```
async func fetch_data(str url) -> str {
    time_sleep(100)
    return "data from " + url
}

async func main_async() {
    var result = await fetch_data("https://example.com")
    printin(result)
}

main {
    run_async main_async()
}
```

### Implementation
- Add `TOKEN_ASYNC`, `TOKEN_AWAIT`, `TOKEN_RUN_ASYNC` keywords
- Add `AST_ASYNC_FUNC_DECL`, `AST_AWAIT_EXPR`, `AST_RUN_ASYNC` to AST
- Async functions compile to generator-style state machines
- `await` compiles to: suspend coroutine, register continuation
- `run_async` starts the event loop (simple cooperative scheduler)
- Runtime: task queue + simple round-robin scheduler

### Files to modify
- `include/lexer.h` — add tokens
- `src/lexer.c` — recognize keywords
- `include/ast.h` — add AST nodes
- `src/parser.c` — parse syntax
- `src/semantics.c` — validate await only inside async
- `src/codegen.c` — compile async as coroutines
- `include/runtime.h` — add async runtime (task queue, scheduler)

## Feature 3: Multithreading

Platform-aware threading with `thread`, `Mutex`, `Channel`.

### Language Syntax
```
import thread

var m = mutex_new()
var ch = channel_new()

thread spawn {
    mutex_lock(m)
    // critical section
    mutex_unlock(m)
    channel_send(ch, "done")
}

var msg = channel_recv(ch)
```

### Implementation
- Add `TOKEN_THREAD`, `TOKEN_SPAWN` keywords
- Runtime header provides `cnext_thread_spawn()`, `cnext_mutex_*()`, `cnext_channel_*()` as C wrappers
- On Windows: use Win32 threads (CreateThread, CRITICAL_SECTION)
- On POSIX: use pthreads
- Codegen: `thread spawn { ... }` generates a thread creation call
- Mutex/Channel are runtime structs with platform-specific implementations

### Files to modify
- `include/lexer.h` — add tokens
- `src/lexer.c` — recognize keywords
- `src/semantics.c` — register thread module functions
- `src/codegen.c` — generate thread/mutex/channel C calls
- `include/runtime.h` — add thread runtime (platform-specific)

---

## Implementation Order

1. **Coroutines first** — extends existing generator infrastructure, lowest risk
2. **Async/Await second** — builds on coroutines
3. **Multithreading last** — independent, can be done in parallel
